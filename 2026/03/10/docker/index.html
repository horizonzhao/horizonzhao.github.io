docker基础

Posted on 2026-03-10 In 技术分享
容器化部署运维。

1. 微服务

  • 服务拆分：将单体应用拆分成多个小型的服务，每个服务专注单一业务
  • 独立性：每个服务可以独立开发、测试、部署和扩展
  • 每个服务可以用不同代码语言

2. Docker 容器化部署运维

2.1 基本概念

  • Docker：打包【程序 + 环境】
  • 环境中最重要：操作系统
  • Dockerfile：就是一个 to-do list

2.2 Dockerfile 示例

说明：这只是描述要做的事，需要在命令行输入 docker build。接着，docker 会依据 Dockerfile 一行行配置好【程序 + 环境】，打包成【容器镜像】（类似压缩包）。

2.3 Docker Registry

Q：容器镜像如何发给多台服务器呢？

类似 git push/pull：

  • docker push
  • docker pull
  • docker run —— 才真正启动容器

基于此可以搭建官方 / 私有镜像仓库：

  • 官方：DockerHub
  • 非官方：清华 tuna
  • 各公司内部也有自己的镜像仓库

2.4 容器特性

程序运行在独立环境中，互相隔离，基于 docker 软件，依托宿主机操作系统运行。

2.5 Docker Compose

docker-compose.yml：多个容器组成的一整套服务

想部署多个容器，且对容器顺序有要求，不要挨个 docker run，而是写入 docker-compose.yml 文件中。

3. Kubernetes (k8s)

以 API 编程的方式管理安排各个容器的引擎。
